Where the data lives

Two sources, both on your Mac

Claude Code is already writing forensic detail about every session you run. You just don't have anything reading it.

ALWAYS ON

Session JSONLs

One file per session. Each line is a JSON event — every tool call, every token, every message.

~/.claude/projects/*/*.jsonl
  • user / assistant messages (with token usage)
  • tool_use + tool_result pairs (latency comes from here)
  • session result: duration, errors, model
OPT-IN

OTEL telemetry

Real-time event firehose. POSTed to any URL you specify. Cleaner than JSONL. Opt-in via settings.json.

POST /v1/logs · /v1/metrics
  • tool_result · api_request · api_error
  • hook_execution_start / complete
  • compaction · tool_decision · MCP attribution

Your laptop is both the source and the sink

Plain-english

What "OTEL" actually is

For the non-observability folks.

OpenTelemetry (OTEL) is the standard most developer tools use to emit "what's happening right now" events. It's a spec — a shape for the data — and an endpoint format (POST /v1/logs, /v1/metrics).

Claude Code supports it. You tell Claude Code to send telemetry to any OTEL endpoint you want. The dashboard on your laptop is an OTEL endpoint. You point Claude Code at http://localhost:8765, and suddenly you're reading your own agent's firehose — no hosted service, no enterprise plan, no paywall between you and your data.

Setup · 3 steps

How to enable it on your Mac

The build prompt ships a helper that does this for you, backing up your settings first. Here's what's actually happening under the hood.

1

Add six env keys to ~/.claude/settings.json

These turn on telemetry and point Claude Code at your local dashboard. The setup helper backs up the file first and only adds keys that aren't already there — it never overwrites your own config.

2

Quit Claude Code and reopen it

Env vars only apply to new sessions. Any existing sessions won't emit telemetry until they restart. One-time chore, then you're done.

3

Run a prompt — events start flowing in ~30 seconds

Open http://localhost:8765. First load might show empty panels for a few seconds — that's the initial JSONL sync. Then your live data lights up.

{ "env": { "CLAUDE_CODE_ENABLE_TELEMETRY": "1", "OTEL_EXPORTER_OTLP_ENDPOINT": "http://localhost:8765", "OTEL_EXPORTER_OTLP_PROTOCOL": "http/json", "OTEL_METRICS_EXPORTER": "otlp", "OTEL_LOGS_EXPORTER": "otlp", "OTEL_LOG_TOOL_DETAILS": "1" } }
Don't edit by hand — the build prompt instructs the agent to generate a setup script that backs up your settings.json before writing, and only adds missing keys. It never overwrites anything you've already set.
